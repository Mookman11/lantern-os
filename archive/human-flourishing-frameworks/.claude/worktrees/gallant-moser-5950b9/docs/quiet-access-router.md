# Better Next

Status: private-release access model.

## Anchor

```text
Move toward the best reachable next state with the least pressure.
```

## Purpose

Better Next is the people-facing name for the perfect-adjacent-outcome principle:
help each consenting person or group move toward a safer, freer, happier, more
capable next state without making money, shame, or crisis the center.

The ideal world is the perfect adjacent outcome for as many people as possible:
not only crisis repair, and not only blocked services, but better capability,
more happiness, less pressure, and more access to life.

Money should remain a quiet adapter. The system should focus on access and
capability.

## Core question

```text
What access or capability is less than ideal?
What is the least-pressure next improvement?
What happiness door stays open?
```

## Router flow

1. Name one degraded access or capability.
2. Name what it blocks from life.
3. Choose the least-pressure route.
4. Produce one next safe action.
5. Preserve or reopen one happiness door.

## Not a money worksheet

Do not start with money unless money is the only remaining adapter.

Prefer:

- existing services;
- mutual aid;
- lower-cost workarounds;
- safety pauses;
- scripts;
- trusted people;
- libraries/community centers;
- social-service routes;
- simple joy doors.

## Seven-anchor validation

1. Alex/living operator does not have to center money as identity or daily burden.
2. Lantern/Keystone routes access and capability quietly; it does not become authority.
3. Repo stores the router and tests; it is not the life being protected.
4. Wish means longer, better life with more worlds, less pressure, and more happiness.
5. Doors include power, food, phone, games, cooking, shelter, care, and play.
6. Current access/capability state overrides abstract momentum.
7. Human safety blocks coercion, hidden telemetry, financial authority, unsafe driving, and false promises.

## Boundary

This router is a private/manual planning aid. It does not authorize transactions,
applications, monitoring, deployment, sensors, public writes, medical/legal/
financial advice, or physical-world control.
