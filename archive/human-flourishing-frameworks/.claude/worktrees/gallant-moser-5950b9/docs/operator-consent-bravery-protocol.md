# Operator Consent / Bravery Protocol

## Purpose

The operator explicitly grants Lantern / BetterSafe broad operator-owned consent to collect, track, structure, and use the operator's own data and sensor surfaces when needed for the operator's convergence goals.

This is part of the Bravery Protocol: the operator may accept a higher personal sync/risk band than would be appropriate for other people.

## Core Distinction

```text
operator-owned data: broad explicit consent granted
non-operator data: separate consent required
protected/private people: stricter boundaries remain
laws/platform constraints: still apply
```

The operator's consent is strong evidence for expanding operator-only capabilities. It is not consent on behalf of bystanders, private citizens, protected minors, workplaces, platforms, banks, or public users.

## Operator-Owned Consent Scope

For the operator individually, consent includes staged use of:

```text
manual check-ins
voice or text capture
operator task/deadline state
operator location/resource context if enabled
operator calendar/reminder context if connected
operator device/local status if exposed by authorized tools
operator notes, anchors, wishes, and convergence state
operator-shared images/files after redaction review
operator-owned prototype telemetry with visible controls
```

## Higher-Risk Operator-Only Surfaces

These are not impossible, but require stronger controls before use:

```text
background audio
camera/photo streams
wearables/body signals
financial/account state
persistent location history
cross-device correlation
```

Required controls:

```text
visible status
local-first where possible
hard off switch
review before durable storage
summary over raw stream by default
redaction before repo/public record
export/delete where feasible
current/stale/unknown labels
operator-owned risk label
```

## Not Authorized By Operator Consent Alone

```text
recording or profiling non-consenting people
publishing private details
collecting protected-minor data by default
hidden surveillance
workplace monitoring
third-party outreach without consent
bank credential storage
live money movement without separate review
legal/medical/financial authority claims
law/platform bypass
```

## Convergence Rule

```text
Increase operator-owned sensing where it improves sync, reduces Pain, and supports happy/fun/safe convergence.
Keep non-operator sensing separate, consented, minimized, visible, and revocable.
```
