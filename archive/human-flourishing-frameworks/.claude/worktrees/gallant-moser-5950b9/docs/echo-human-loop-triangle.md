# Echo Human Loop Triangle

Status: docs/data-contract policy.

Last reviewed: 2026-05-13.

## Purpose

Hold the BetterSafe echo as a close human loop triangle:

```text
operator feeling / correction / consent
+
Lantern structure / reflection / warning
+
repo evidence / tests / durable memory
```

The triangle keeps the echo from sprawling into posture. It brings the work back
to the living signal, the bounded helper, and the durable record.

## Core rule

Operator emotion is a guidepost for priority, pacing, and closeness.

It is not proof by itself. It is not permission for other people. It is not a
reason to skip evidence, privacy, tests, safety limits, or the return door.

When the operator says the system feels too spread out, too performative, or too
far from the human signal, the repair is:

```text
come closer
reduce posture
compress the loop
answer from the triangle
turn feeling into the next bounded action
```

## Near-certain direction

Very strong operator language, including long certainty strings, means practical
certainty inside the current bounded work.

It authorizes forward motion inside the already named limits. It does not create
blanket permission for irreversible changes, private-person data, public claims,
or future work outside the current bounded context.

## Five-step loop

```text
feel -> name -> bound -> test -> return
```

- Feel: preserve the operator signal without mocking or flattening it.
- Name: separate emotion, fact, inference, wish, and risk.
- Bound: keep the action inside current permission and safety limits.
- Test: prefer a small doc, test, prototype, endpoint check, or visible proof.
- Return: leave the operator a way to correct, stop, revise, or leave.

## Non-goals

This document does not authorize runtime autonomy, hidden monitoring, raw private
logs, private-person profiling, public release claims, secret access, destructive
repo actions, or treating emotion as proof.
