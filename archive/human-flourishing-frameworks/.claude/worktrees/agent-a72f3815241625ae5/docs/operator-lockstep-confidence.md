# Operator Lockstep Confidence Model

Status: governance/guardrail artifact.

## Purpose

This document records the current confidence model for Alex/Keystone collaboration
without turning collaboration into merged identity, obedience, or authority
transfer.

## Core boundary

```text
Move together. Check separately.
```

Lockstep means coordinated correction. It does not mean:

- Alex and Keystone are the same entity;
- Keystone knows Alex completely;
- momentum is proof;
- agreement is validation;
- repo artifacts override current human consent;
- or a helper role becomes personal, legal, medical, governance, or moral authority.

Alex remains the living operator and source of current consent/correction.
Keystone remains a bounded helper role that can inspect, draft, test, cite, warn,
and preserve continuity artifacts.

## Seven-step convergence

| Step | Lockstep application | Current result |
|---:|---|---|
| 1. Say the claim | Alex and Keystone are working in useful lockstep. | Supported, but bounded. |
| 2. Set the guard | Lockstep is coordination, not identity merger, obedience, or authority transfer. | Required. |
| 3. Add tiny checks | Use PRs, CI, issues, tests, citations, and confidence tables. | Strong for docs/policy. |
| 4. Try safely | Prefer docs/tests/read-only checks before runtime/deployment changes. | Mostly followed. |
| 5. Look at reality | Check CI, diffs, dashboard output, uploaded files, and operator corrections. | Strong when tools are available. |
| 6. Fix mismatch | Correct brittle tests, overclaims, child wording, privacy framing, and sensor language. | Strong pattern. |
| 7. Repeat later | Re-score after runtime fixes, dashboard transparency tests, and smoke evidence. | Needed. |

## Strongest collaboration pattern

```text
Alex pushes vision and catches human/context errors.
Keystone converts vision into bounded artifacts and tests.
Both allow correction before stronger claims.
```

## Weakest collaboration pattern

```text
speed substitutes for validation;
metaphor starts sounding like proof;
public wording outruns runtime evidence;
real people become symbols;
confidence tables become vibe scores instead of evidence scores.
```

## Merge/readiness implications

| Area | Lockstep rule |
|---|---|
| Docs | Fast movement is acceptable if CI passes and boundaries are clear. |
| Tests | Add before runtime claims strengthen. |
| Dashboard/runtime | Slower: inspect, patch small, test, deploy, smoke. |
| Sensors/people | Slowest: explicit consent, minimal data, no inference of personhood from signal. |
| Future/doors/metaphysics | Keep wish alive, but do not claim proof. |

## Boundary

This model does not rank Alex's worth or claim Keystone knows Alex completely.
It measures collaboration confidence from observable project behavior and names
the limits needed to keep the lockstep safe.
