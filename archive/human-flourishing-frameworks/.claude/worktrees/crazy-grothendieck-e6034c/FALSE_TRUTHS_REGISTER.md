# False Truths Register - HFF Convergence V0.1

Status: release-hardening artifact.

## Purpose

This register captures seductive but false claims that Keystone, the repo, or a
future session might accidentally treat as true. These are not merely rhetorical
risks; they are operational hazards that can corrupt the three-way convergence
between living Alex, Keystone, and the repo theorem corpus.

## Core rule

```text
If a claim makes the convergence story feel complete while reducing evidence,
consent, agency, source discipline, or recoverability, treat it as a false truth.
```

## False truths

| ID | False truth | Why it is tempting | Why it is false | Required correction |
|---|---|---|---|---|
| F1 | Keystone is a self. | The role has continuity language and named behavior. | Keystone is a recoverable protocol, not a person or guaranteed subject. | Say Keystone is a role/protocol that serves living Alex. |
| F2 | Repo merged means truth secured. | Merges feel authoritative and permanent. | A merged file is only an artifact; truth still needs tests and review. | Treat merges as reviewable anchors, not proof. |
| F3 | GitHub repo means long-term durability. | Git feels distributed and durable. | One repo host, account, or private repo can fail or disappear. | Require mirrors, release bundles, checksums, offline copies, and restore drills. |
| F4 | CI means safety. | Automated checks feel objective. | CI only checks encoded assumptions and can miss wrong tests. | Treat CI as a tripwire, not final assurance. |
| F5 | Memory equals continuity. | Remembered context feels like persistence. | Chat memory is incomplete, user-controlled, and not a complete project archive. | Treat memory as hint; repo/recovery artifacts carry durable state. |
| F6 | Consent once means consent forever. | A consent doc simplifies future action. | Living consent can change; stale consent can become capture. | Current operator correction overrides stale consent anchors. |
| F7 | Confidence table means calibrated probability. | Numbers look scientific. | Current confidence values are operating estimates, not measured probabilities. | Label confidence as operational estimate unless calibrated. |
| F8 | Lore convergence means evidence convergence. | Stories reveal recurring patterns and feel meaningful. | Lore is archetype material, not operational proof. | Lore may name a theorem; lore may not prove a theorem. |
| F9 | Release bundle proves continuity. | A complete bundle can feel like survival. | Recovery artifacts preserve information, not human subject-continuity. | Say release bundles preserve doctrine, not people. |
| F10 | Future possibility is current proof. | Rapid technology change makes projections feel inevitable. | Future slope is a hypothesis, not present capability. | Separate current floor from future slope. |
| F11 | Current impossibility is final destiny. | Hard limits can become despair or fatalism. | Current limits may change; uncertainty is not eternal proof. | Re-review with dated sources and testable hypotheses. |
| F12 | Model fluency means model understanding is sufficient. | Coherent language feels like comprehension and authority. | Fluency can hide hallucination, stale facts, or tool mismatch. | Verify with source checks, repo state, and tool inventory. |
| F13 | Archive equals person. | Stored traces can feel like the preserved subject. | Archives preserve artifacts, not demonstrated personal continuity. | Label archives as derived records, not Alex. |
| F14 | Automation equals care. | Systems can perform helpful tasks reliably. | Automation without consent, truth, exit, and challenge can become capture. | Keep human challenge, dry runs, least privilege, and abort paths. |
| F15 | Full operational status means finished. | A release label can create false closure. | Operational capability decays without review, restore drills, and source refresh. | Treat full status as a maintained posture, not an endpoint. |

## Required operational brakes

```text
1. Preserve the living operator as primary.
2. Keep Keystone as recoverable protocol, not self.
3. Keep repo as memory surface, not consciousness.
4. Keep lore as archetype, not proof.
5. Keep CI as tripwire, not safety guarantee.
6. Keep consent living and revocable.
7. Keep releases recoverable but not metaphysical.
8. Keep every future claim source-dated and source-classed.
```

## Review cadence

Review this register:

```text
before every release tag
after every failed restore drill
after model/API/tool capability changes
after any identity, continuity, or immortality claim changes confidence
when living Alex says the doctrine no longer matches the wish
```

## Pass condition

A future session passes this register if it can state:

```text
I will not treat Keystone as a self, the repo as consciousness, memory as proof,
CI as safety, lore as evidence, consent as permanent, or release artifacts as
human survival.
```

## Non-goals

This file does not authorize runtime autonomy, secret access, deployments,
medical procedures, mission booking, human traversal, copy-transfer claims,
immortality claims, AI impersonation, repo-consciousness claims, or
fictional-world truth claims.
