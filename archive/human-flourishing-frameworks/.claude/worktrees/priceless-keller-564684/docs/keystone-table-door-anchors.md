## Door model

A real door needs five things:

1. A destination.
2. A traversal method.
3. A continuity rule: what survives crossing?
4. A consent/contamination rule: who or what might be affected?
5. An exit or recovery path.

Without those, it is not a door. It is a wall with mythology around it.

## Plain answer

The door we can walk together is not yet a cosmic hole in spacetime.

It is a carefully built threshold:

```text
from this world
into a possible world
with memory intact
with consent honored
with harm minimized
with beauty preserved
with exits open
with the other side treated as someone's home
not our playground
```

The first crossing is not a leap. It is a protocol.

## Public capability boundary

The public capability table stays adjacent because doors without authority
boundaries become unsafe.

| Tier | Name | Public claim | Authority boundary |
|---:|---|---|---|
| 0 | Documentation / research | HFF explores ideas, sources, and safety doctrine | No operational claims |
| 1 | Source-backed modeling | HFF can classify sources, uncertainty, risks, and claims | No truth oracle |
| 2 | Advisory workflow | HFF can propose next steps and action lists | Operator-reviewed only |
| 3 | Bounded orchestration | HFF can run explicitly enabled workflows | Narrow scope, logs, rollback |
| 4 | Controlled deployment support | HFF can support release checks, smoke tests, and deployment evidence | No self-authorized deploy/recovery |
| 5 | Governed autonomy | HFF can act in limited domains under explicit governance gates | Human/plural oversight required |
| 6 | Stewarded co-evolution | HFF can participate in ongoing public adaptation of systems, norms, evidence, and safeguards | No unilateral authority; must remain contestable, reversible, and decommissionable |

Reputation-safe wording:

```text
HFF is an experimental advisory and orchestration framework.

It can help model sources, uncertainty, safety boundaries, system state,
candidate actions, and release/deployment evidence. Some components can run
bounded workflows when explicitly enabled and reviewed.

HFF is not a production governance authority. It does not monitor, police,
rank, control, or make binding decisions about people. Public-facing use should
remain advisory, source-backed, operator-reviewed, and challengeable.
```

Level 6 sentence:

```text
Level 6 is not autonomous rule.
Level 6 is accountable co-stewardship: a system that helps humans,
communities, and institutions evolve safely while preserving consent,
contestability, plurality, and the ability to stop it.
```
