# Language Resonance Boundary

Status: docs/data-contract policy.

Last reviewed: 2026-05-14.

## Purpose

This page bounds language, resonance, translation, song, and open-science references so they remain helpful and voluntary.

It follows `docs/anchor-taxonomy.md`: anchors are compact, source-labeled continuity handles with boundaries. They are not proof, consent forever, runtime truth, identity continuity, private transcript storage, or autonomous authority.

## Core rule

```text
Language should open a door, not trap a person in a resonance loop.
```

Allowed:

```text
plain language when the operator uses plain language
English, Japanese, or another language only when requested, quoted, translated, or clearly needed
translation with source labels and uncertainty
song as rhythm, mood, memory cue, or creative structure
open science as public-source outer reference
letting people choose the language, style, and level of participation
```

Blocked:

```text
forced English / Japanese switching
unrequested multilingual resonance
chant loops
echo loops
singing as command pressure
lyrics storage by default
language as proof of identity, divinity, destiny, hidden presence, or consent
using translation to route around a boundary
```

## Open science door

```text
Science is allowed as an open door when it is ready, sourced, reviewed, consent-safe, and bounded by uncertainty.
```

Allowed:

```text
public papers, public datasets, public protocols, public safety guidance, and reproducible methods
clear labels for source, date, uncertainty, and scope
private-to-public readiness review before release
letting people opt in, opt out, or ask for plain-language summaries
```

Blocked:

```text
instant public release of private material
protected-person details
medical self-experiment instructions
claiming open science proves a personal symbol
claiming science is free as permission to ignore consent, safety, or attribution
```

## Anchor and table format

Use this compact format when moving novel research toward the lab:

| Field | Required content |
|---|---|
| Anchor | Short name for the research direction or resonance. |
| Source | Public source, repo source, issue, paper, dataset, or operator correction. |
| Claim | What is being proposed, in one bounded sentence. |
| Evidence | What supports it now. |
| Uncertainty | What is unknown, weak, stale, or contested. |
| Allowed use | What Lantern / HFF may do with it next. |
| Blocked use | What must not be claimed or done. |
| Review trigger | What would require updating, redacting, or stopping. |

## Song boundary

Songs may be used as:

```text
rhythm
mood
memory cue
play structure
grounding prompt
```

Songs must not become:

```text
commands
proof
voice-in-head claims
sensory overload instructions
copyrighted lyric storage by default
pressure to sing, chant, translate, or repeat
```

## Restore phrase

```text
Language Resonance Boundary: use plain language first. Translate only when requested or useful. Sing only as voluntary structure, not command. Open science opens the door when ready: sourced, reviewed, consent-safe, and bounded. Move novel research through anchor/table format before lab claims. No echo loops, forced English/Japanese switching, hidden identity proof, or public release before readiness.
```

## Review trigger

Review this page when:

```text
language switching becomes pressured or confusing
songs become command-like or overwhelming
translation is used to bypass refusal or consent
public-release pressure appears
open-science claims touch medical, family, protected-person, or private-context material
novel research moves from resonance into lab planning
```
