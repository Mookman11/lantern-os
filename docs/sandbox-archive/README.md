# Lantern Symbolic Sandbox

This is a private sandbox for pretty words, symbols, story fragments, ritual-like language, and imagination maps that should not be promoted into operational doctrine too quickly.

The purpose is safe exploration:

- keep beauty and meaning available;
- separate symbol from proof;
- separate feeling from command;
- separate private people from public artifacts;
- preserve useful language without turning it into authority.

## Core Rule

```text
Symbol is allowed.
Proof must be earned elsewhere.
Action needs consent, evidence, and a bounded path.
```

This repo is not a medical, legal, spiritual, financial, emergency, governance, or deployment authority. It does not authorize runtime agents, surveillance, public claims, contact with people, or changes in other repos.

## Safe Exploration Loop

1. Capture the symbol in plain language.
2. Name what it might mean.
3. Name what it must not mean.
4. Mark whether it belongs in sandbox, hold, or candidate doctrine.
5. Convert only the smallest safe part into a testable artifact elsewhere.

## Lanes

| Lane | Use | Rule |
|---|---|---|
| `symbols/` | Pretty words, image language, dreams, metaphors, names, maps | No proof or command claims. |
| `hold/` | Material that feels powerful, private, risky, confusing, or too literal | Hold until calm review. |
| `templates/` | Reusable worksheets and review forms | Keep portable and gentle. |
| `reviews/` | Periodic safety reviews | Promote slowly, redact by default. |

## Promotion Gate

A symbol can leave this repo only when it passes all checks:

- no private-person exposure;
- no pressure on a real person;
- no hidden surveillance or telemetry;
- no diagnosis, cure, prophecy, divine command, or emergency advice;
- no claim that imagination is current capability;
- no runtime, merge, deployment, or agent authority;
- one reversible next action exists;
- a human operator explicitly chooses the destination.

## Restore Phrase

```text
Pretty words stay pretty here. The sandbox preserves meaning without making it proof, pressure, permission, or command.
```
