# BETTER iPhone App Concept

Status: future product contract. No native iPhone app, App Store listing, or
background sensor permission is claimed yet.

BETTER is the future iPhone shell with two modes:

| Mode | Purpose | Boundary |
|---|---|---|
| BetterSafe | Organize safety, access, documents, support, and next-step packets. | Not medical, legal, financial, emergency, surveillance, child-facing, or autonomous authority. |
| BetterFun | Play, learn, make, explore, and recover joy without turning fun into extraction. | No gambling framing, fake ROI, hidden profiling, dark patterns, or collection of private child/family/social graph data. |

Start PWA/Shortcut-first: Safari Add to Home Screen, explicit Shortcut sends,
LAN/VPN/local relay before public tunnels, and no default native permissions,
background collection, contact scraping, health data pull, location tracking, or
account access.

Each user bundle should be compressed and prebundled with profile boundaries,
BetterSafe packets, BetterFun preferences, validated local/cloud URLs, RAG
evidence labels, correction history, and a visible rollback receipt.

The split-brain idea is only an engineering metaphor. BetterSafe and BetterFun
may specialize, but they must communicate through one protocol:

```text
observe -> record -> compare -> propose -> human approve -> apply -> verify -> repeat
```

If modes disagree, Lantern One IDE shows both claims, the evidence for each,
the limit, and the next reversible action.
