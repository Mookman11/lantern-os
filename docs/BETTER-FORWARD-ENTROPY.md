# BETTER Forward Entropy Study

Status: convergence study note.

Forward entropy is the natural direction of an unattended system: repos drift,
dashboards split, tunnels go stale, agents repeat old context, commands fork,
and cash claims blur unless the system spends a little force to keep truth
ordered.

## What To Study

| Entropy path | Small reversing force |
|---|---|
| Local and cloud URLs disagree. | Poll both and show the mismatch. |
| MCP docs advertise tools that are not exposed. | Trust `tools/list` and validation receipts. |
| Agents repeat stale plans. | Route commands through `POST /api/command`. |
| BETTER modes fork private state. | Use one protocol and show both claims. |
| Revenue claims inflate. | Keep manual invoice separate from cleared cash. |
| Symbol becomes command. | Return it to sandbox boundary and one reversible action. |

## Butterfly Rule

Solve big problems with little forces:

```text
one observation
one receipt
one command route
one reversible fix
one rollback path
```

The point is not to deny entropy. The point is to study its forward path early
enough that one small verified move can redirect it before it becomes a tidal
mess.

## BETTER Test

A BETTER-aligned change must answer:

- What entropy is increasing?
- What is the smallest force that reverses it?
- What evidence proves the force worked?
- What rollback keeps the move humane?
- What stays symbolic, local, or held?
