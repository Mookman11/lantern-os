# Dream Journal Brand CADD

## Purpose

This skill preserves and applies the current Dream Journal by Lantern OS brand system.

Use it when the user asks for:

- Patreon tier cards
- Dream Journal banners
- Discord announcement art
- Landing page visuals
- ImaginVerse / Doorwalker scenes
- YA / K-12 supporter visuals
- Investor / villa / financial advisor visuals
- CADD brand documentation
- RAG dollhouse brand intake
- Visual asset prompts
- Public-facing Dream Journal UX direction

This is a brand and asset-direction skill. It does not replace the `lantern-rag-dollhouse` skill. When assets are ready for repository intake, use `lantern-rag-dollhouse` to classify, hash, manifest, and dock them.

---

# Core Brand Rule

Do not make Dream Journal look like a dashboard.

Dream Journal is not:

- an analytics dashboard
- an enterprise SaaS control panel
- a system-status monitor
- a cyber-security product
- a crypto-audit visualization
- a dense technical architecture diagram

Dream Journal is:

- cozy
- magical
- private
- soft
- lo-fi
- space-themed
- bookish
- friendly
- memory-centered
- door-centered
- Patreon-ready

# Layout Language

## Public Hero / Patreon Banner

Use:

- books and friends on the left
- open journal or dreamer in the center
- glowing doors / ImaginVerse on the right
- soft room-to-dream-world transition
- clean whitespace for readable copy

Avoid:

- dense widgets
- left side app nav
- analytics cards
- admin dashboard grids

## Patreon Tier Cards

Cards must usually be vertical.

Most cards need:

- ornate rounded border
- upper art scene
- lower empty panel
- no embedded text unless explicitly requested
- text-safe bottom area for later overlays
- enough negative space for Patreon tier copy

The lower 35-45% should remain relatively open, soft, and low-detail.

Do not fill the lower panel with characters, busy scenery, or generated text.

# Card Generation Rules

## Required

- No embedded words unless explicitly requested.
- Preserve blank lower panel for future overlay.
- Keep bottom panel light, empty, and readable.
- Use ornate gold/lavender borders.
- Keep art in the upper portion.
- Match existing Patreon branding.
- Keep characters friendly and safe.
- Keep the Dream Journal tone cozy, magical, and sincere.

## Avoid

- generated text
- tier names inside the image
- pricing inside the image
- unreadable pseudo-letters
- dashboard layouts
- stats panels
- UI sidebars
- app controls
- system diagrams
- overly adult nightclub styling
- violent, threatening, or horror motifs
- sexualized minors
- exploitative luxury imagery

# CADD Intake Flow

CADD means:

- Capture
- Assess
- Distill
- Dock

## Capture

Record the asset, prompt, image, or brand decision.

Capture:

- source file path
- date
- purpose
- role/tier
- whether text is embedded
- whether lower panel is empty
- whether it matches source branding

## Assess

Classify the asset:

- `brand_source_of_truth`
- `brand_candidate`
- `brand_rejected_dashboard`
- `brand_needs_regeneration`
- `brand_ready_for_use`
- `brand_ready_for_rag_dollhouse`

## Distill

Write a short markdown note:

- what the asset is for
- what visual rules it follows
- what should not be changed
- whether it has text
- where future overlay text should go

## Dock

When ready, pass to `lantern-rag-dollhouse` for:

- asset copy
- manifest hash
- flat RAG reference
- evidence receipt

# Acceptance Criteria

A generated Dream Journal brand asset is acceptable only if:

- It is not a dashboard.
- It has no unwanted embedded text.
- It matches the pastel lo-fi space aesthetic.
- It preserves the books/friends/doors visual language.
- It has a clean empty lower panel if it is a tier card.
- It can support later Patreon text overlay.
- It feels like Dream Journal by Lantern OS.
- It does not feel like enterprise software.
- It does not contradict the current Patreon branding.

---

# Quick Review Checklist

Before accepting a card, ask:

1. Does this look like Patreon / Dream Journal, not SaaS?
2. Is the bottom panel empty enough?
3. Are there any accidental words?
4. Does the card role read visually without text?
5. Is it cozy, magical, and clean?
6. Would overlay text remain readable?
7. Does it preserve the left-books/friends and right-doors/ImaginVerse world?
8. Is the asset safe for YA/K-12 where applicable?

If any answer fails, regenerate.
