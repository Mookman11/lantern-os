# Code of Conduct

Lantern OS work should be careful, direct, and evidence-led. Contributors and
agents are expected to preserve user work, respect privacy, and report what is
real without overstating verification.

## Expected Behavior

- Be respectful and specific.
- Preserve private data and local-only context.
- Ask for clarification when a change could damage user work.
- Prefer small, reviewable changes with clear validation.
- State uncertainty plainly.

## Unacceptable Behavior

- Sharing secrets, private journal data, or local receipts without permission.
- Claiming a service, route, MCP tool, or agent workflow is live without
  verification.
- Resetting, deleting, force-pushing, or overwriting work without explicit
  permission.
- Harassment, threats, or abusive communication.

## Enforcement

The repository owner may remove contributions, revoke access, or require
remediation for behavior that violates this code.
