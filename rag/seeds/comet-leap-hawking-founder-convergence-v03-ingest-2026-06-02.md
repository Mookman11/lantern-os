## Section 4: Promoted Architecture — Human Destination Workbench

The core architecture is a reproducible local loop that works when Canva fails, when cloud APIs are unavailable, and
when external models are not trusted.

```
Operator intent → repo/RAG dollhouse → evidence ledger → Bayesian confidence → local renderer suite
→ tables/graphs/spectrograph/tesseract/BW01 → Markdown/PDF packet → optional export adapters
```

| Layer | Input | Local output | Boundary |
|---|---|---|---|
| Repo/RAG layer | Lantern OS files, reports, uploaded sources, asset manifests | retrievable evidence pack | do not pretend metadata-only repos are cloned |
| Evidence layer | claims, source paths, web/official sources, uploaded source notes | confidence and decision labels | projection never overrides verified state |
| Renderer layer | physics parameters, data arrays, audio state, diagrams | SVG/PNG/GIF/WAV/PDF | no external generator dependency |
| Action layer | founder decision and next sprint tasks | P0-P4 backlog and validation checklist | no repo write without exact approval |
| Adapter layer | finished local artifact | Canva copy, GPT summary, cloud export | adapter failure cannot halt core work |
